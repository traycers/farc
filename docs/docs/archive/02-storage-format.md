# Формат хранилища на диске

```
Тогда текущее состояние:

```
┌──────────────────────────────────────┐
│ Пролог                               │
│ ┌─ Фиксировано навсегда ───────────┐ │
│ │  · магическое число               │ │
│ │  · время каталога (ns)            │ │
│ │  · версия формата                 │ │
│ │  · размер пролога                 │ │
│ │  · кол-во записей в каталоге      │ │
│ │  · размер каталога                │ │
│ ├─ Зависит от версии ──────────────┤ │
│ │  · параметры хранилища ...        │ │
│ └──────────────────────────────────-┘ │
├──────────────────────────────────────┤
│ Каталог (переменная длина)           │
│   для каждого фблока:                │
│     · UUID                           │
│     · состояние / флаги              │
│     · begin_time (ns)                │
│     · end_time (ns)                  │
│     · кол-во каналов (uint8)         │
│     · каналы [64] (uint16, фикс.)    │
│       неиспользованные = 0           │
├──────────────────────────────────────┤
│ CRC32 заголовка (пролог + каталог)   │
├──────────────────────────────────────┤
│ Данные (payload)                     │
├──────────────────────────────────────┤
│ TOC (переменная длина)               │
│   (непрозрачно для Storage)          │
├──────────────────────────────────────┤
│ Эпилог (фиксированный размер)       │
│   · CRC32 данных                     │
│   · CRC32 TOC                        │
│   · размер TOC                       │
└──────────────────────────────────────┘
```

Каждая запись каталога теперь фиксированного размера: UUID(16) + состояние/флаги(1) + begin(8) + end(8) + кол-во каналов(1) + каналы(128) = **162 байта**. При 10 000 фблоках — ~1.6 МБ каталог. Парсинг тривиальный — прыгаешь по записям с постоянным шагом.
```

## 1. Общая структура

Хранилище (файл или блочное устройство) состоит из последовательности фблоков одинакового размера.

Первый фблок (индекс 0) содержит суперблок и копию индексов. Остальные фблоки содержат либо данные потребителя, либо дополнительные копии индексов.

Логическая структура:

- Смещение 0: суперблок (4 КБ)
- Далее: фблоки, идущие подряд, от индекса 0 до N-1
- Каждый фблок занимает ровно `fblock_size` байт
- Физическое смещение фблока с индексом i: `i * fblock_size`

## 2. Суперблок

Суперблок — это первый сектор хранилища. Размер: 4096 байт (4 КБ).

Суперблок содержит всю информацию, необходимую для интерпретации остального содержимого хранилища: размеры, версию формата, расположение копий индексов.

### 2.1. Поля суперблока

| Смещение | Размер | Поле | Описание |
|----------|--------|------|----------|
| 0 | 8 | magic | Сигнатура "FBLOCK01" (в байтах: 0x46 0x42 0x4C 0x4F 0x43 0x4B 0x30 0x31) |
| 8 | 2 | version | Версия формата хранилища (текущая = 1) |
| 10 | 1 | flags | Флаги состояния хранилища |
| 11 | 5 | reserved_1 | Выравнивание, заполнено нулями |
| 16 | 8 | fblock_size | Размер одного фблока в байтах |
| 24 | 8 | fchunk_size | Размер одного фчанка в байтах |
| 32 | 8 | total_fblocks | Общее количество фблоков в хранилище |
| 40 | 2 | index_entry_size | Размер одной записи индекса в байтах (всегда 40) |
| 42 | 2 | index_copies | Текущее количество копий индексов |
| 44 | 800 | index_fblocks | Массив из 100 элементов по 8 байт. Каждый элемент — индекс фблока, содержащего копию индексов. Неиспользуемые элементы равны 0. |
| 844 | 8 | active_index_fblock | Индекс фблока с самой свежей (активной) копией индексов |
| 852 | 8 | last_update | Время последнего обновления суперблока (Unix nanoseconds) |
| 860 | 8 | checksum | Контрольная сумма (xxHash64) всех предыдущих полей |
| 868 | 3228 | reserved_2 | Заполнено нулями до 4096 байт |

### 2.2. Флаги суперблока (1 байт)

| Бит | Значение | Описание |
|-----|----------|----------|
| 0 | 0x01 | Чистое завершение работы (clean shutdown) |
| 1 | 0x02 | Было грязное завершение (dirty shutdown) |
| 2-7 | - | Резерв, должно быть 0 |

### 2.3. Обновление суперблока

Суперблок обновляется только при:
- Инициализации хранилища.
- Расширении хранилища (добавлении новых фблоков и копий индексов).
- Чистом завершении работы (сброс флага dirty).

При обычной работе (запись фблоков, обновление индексов) суперблок **не перезаписывается**.

## 3. Индексы

Индексы — это метаданные, описывающие состояние каждого фблока в хранилище.

### 3.1. Структура одной записи индекса

Каждая запись соответствует одному физическому фблоку (по его индексу). Размер записи: 40 байт.

| Смещение | Размер | Поле | Описание |
|----------|--------|------|----------|
| 0 | 16 | uuid | UUID фблока. Нулевой UUID (все байты 0) означает, что фблок свободен |
| 16 | 8 | expiry_time | Время окончания актуальности (Unix nanoseconds) |
| 24 | 8 | write_time | Время записи фблока (Unix nanoseconds) |
| 32 | 1 | flags | Флаги состояния фблока |
| 33 | 1 | index_version | Версия структуры записи индекса (текущая = 1) |
| 34 | 2 | reserved | Выравнивание, заполнено нулями |
| 36 | 4 | checksum | CRC32C всех предыдущих полей записи |

### 3.2. Флаги записи индекса (1 байт)

| Бит | Значение | Описание |
|-----|----------|----------|
| 0 | 0x01 | Фблок записан (содержит валидные данные) |
| 1 | 0x02 | Фблок в процессе записи (не завершён) |
| 2 | 0x04 | Фблок битый (физическое повреждение) |
| 3 | 0x08 | Фблок защищён от перезаписи (read-only) |
| 4-7 | - | Резерв, должно быть 0 |

### 3.3. Полный набор индексов

Полный набор индексов — это массив из `total_fblocks` записей, расположенных последовательно. Размер всего набора: `total_fblocks * 40` байт.

Например, для 100 000 фблоков размер индексов составит 4 000 000 байт (~3.8 МБ).

### 3.4. Копии индексов

Для надёжности хранится несколько копий полного набора индексов. Каждая копия размещается в отдельном фблоке с типом "индексы".

При инициализации создаётся минимум 2 копии, рекомендуется 3-4.

Расположение копий:
- Первые K фблоков (индексы 0..K-1) содержат копии индексов.
- Последние K фблоков (индексы N-K..N-1) содержат ещё K копий.

Где:
- K = количество копий, заданное при инициализации.
- N = общее количество фблоков.

Пример для N=10000, K=2:
- Фблок 0: копия индексов №1
- Фблок 1: копия индексов №2
- Фблоки 2..9997: данные потребителя
- Фблок 9998: копия индексов №3
- Фблок 9999: копия индексов №4

### 3.5. Формат хранения копии индексов в фблоке

Фблок типа "индексы" имеет ту же структуру, что и обычный фблок данных: заголовок (128 байт) + данные.

В поле данных фблока хранится:

| Смещение | Размер | Поле | Описание |
|----------|--------|------|----------|
| 0 | 8 | copy_time | Время создания этой копии (Unix nanoseconds) |
| 8 | 4 | copy_number | Номер копии (1, 2, 3, ...) |
| 12 | 4 | entry_count | Количество записей в копии (должно равняться total_fblocks) |
| 16 | 8 | data_checksum | Контрольная сумма всех последующих записей индексов (xxHash64) |
| 24 | N*40 | entries | Массив записей индексов |

Заголовок такого фблока имеет специальный UUID:
- Младшие 8 байт = 0
- Старшие 8 байт = номер копии (1, 2, 3, ...)

В текстовом представлении: `00000000-0000-0000-0000-000000000001` для копии 1.

### 3.6. Выбор активной копии при старте

При открытии хранилища система:

1. Читает суперблок.
2. Получает список всех индексных фблоков из поля `index_fblocks`.
3. Читает каждую копию индексов из соответствующего фблока.
4. Проверяет целостность каждой копии (контрольную сумму данных и CRC записей).
5. Выбирает копию с максимальным `copy_time`, прошедшую проверку целостности.
6. Загружает выбранную копию в память.
7. Обновляет поле `active_index_fblock` в суперблоке.

## 4. Фблок данных

Фблок данных (тип 0x00) — это основная единица хранения данных потребителя.

### 4.1. Структура

Каждый фблок данных состоит из двух частей:
- Заголовок (128 байт) — служебная информация.
- Полезные данные — данные потребителя (размер = `fblock_size - 128`).

Детальная структура заголовка описана в отдельном документе `04-fblock-format.md`.

### 4.2. Размещение на диске

Фблоки данных размещаются в области между начальными и конечными копиями индексов.

Для N фблоков и K копий:
- Индексы данных: с K до N-K-1 включительно.

## 5. Параметры по умолчанию

| Параметр | Значение | Обоснование |
|----------|----------|-------------|
| Размер фблока | 2 ГБ (2,147,483,648 байт) | Крупный блок для минимизации накладных расходов |
| Размер фчанка | 16 МБ (16,777,216 байт) | Компромисс между количеством операций ввода-вывода и точностью обнаружения ошибок |
| Количество копий индексов | 4 | Две в начале, две в конце для надёжности |
| Размер суперблока | 4 КБ | Один сектор диска |

## 6. Порядок байт

Все многобайтовые числа в структурах на диске хранятся в порядке **little-endian**.

Это обеспечивает переносимость между платформами (x86, x86_64, ARM) без дополнительного преобразования на наиболее распространённых архитектурах.

## 7. Контрольные суммы

| Структура | Алгоритм | Размер | Примечание |
|-----------|----------|--------|------------|
| Суперблок | xxHash64 | 8 байт | Быстрый, хорошее распознавание ошибок |
| Запись индекса | CRC32C | 4 байта | Аппаратное ускорение на современных CPU |
| Заголовок фблока | CRC32C | 4 байта | Аппаратное ускорение |
| Данные фблока | xxHash64 | 8 байт | Быстрый для больших объёмов |
| Копия индексов | xxHash64 | 8 байт | Защита всего массива индексов |
