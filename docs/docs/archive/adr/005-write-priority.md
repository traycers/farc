# ADR-005: Приоритет записи над чтением

**Дата:** 2026-04-17
**Статус:** Принято

> Ранее этот ADR имел номер 006. Перенумерован в 005 для сквозной нумерации.

## Контекст

Нельзя остановить функцию чтения с диска, вызванную в операционную систему. При этом запись имеет абсолютный приоритет и должна начинаться почти немедленно после попадания фблока в очередь записи.

## Решение

Чтение производить небольшими порциями (меньше размера фчанка). В StorageEngine необходимо поддержать чтение запрашиваемых данных порциями, т.е. одна команда на чтение разбивается на несколько вызовов функции операционной системы.

**Обоснование:** запись имеет абсолютный приоритет. При получении команды на запись эта команда должна выполняться почти немедленно, но остановить функцию ОС, выполняющую чтение, нельзя. Поэтому чтение, выглядящее снаружи как один вызов, внутри разбивается на несколько коротких, между которыми проверяется очередь записи.

Размер порции чтения выбирается как компромисс между задержкой прерывания (чем меньше порция, тем быстрее отдаётся диск под запись) и накладными расходами на множество системных вызовов. Конкретное значение фиксируется в документе операций хранилища.
