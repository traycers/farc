# ADR-001: Расположение заголовка фблока

**Дата:** 2026-04-15
**Статус:** Принято

## Контекст

Необходимо хранить метаданные фблока (UUID, срок жизни). Рассматривались несколько вариантов: перед данными (внешний заголовок) и внутри области данных фблока.

## Варианты

### 1. Перед фблоком (внешний заголовок)
- ✔ Точное соответствие заявленному размеру данных.
- ❌ Усложняет расширение хранилища (размер заголовка в версии 2.0 может измениться, сломав расчёт смещений).
- ❌ Требует хранения таблицы смещений.

### 2. Внутри фблока
- ❌ Уменьшает полезную ёмкость фблока на размер заголовка.
- ✔ Предельно простой расчёт позиции: `Index * FixedSize`. Поддерживает изменение размера хранилища.
- ❌ При малом размере фблока индексы могут занимать значительную его часть.

## Решение

Хранить заголовок внутри фблока.

**Обоснование:** приоритет отдан надёжности расчётов геометрии хранилища. Расширение или сужение хранилища предсказуемо, т.к. размер фблоков фиксированный. Список смещений для большого диска занимает слишком много места.
